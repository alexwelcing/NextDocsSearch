## Root Causes Identified

### 1. Phantom Sub-Sitemaps (CRITICAL)

**Problem:** The sitemap index at `/sitemap.xml` references four sub-sitemaps:
- `sitemap-0.xml` — works
- `sitemap-core.xml` — **returns 404**
- `sitemap-articles.xml` — **returns 404**
- `video-sitemap.xml` — works

**Root Cause:** In `next-sitemap.config.js`, `sitemap-core.xml` and `sitemap-articles.xml` are listed in `robotsTxtOptions.additionalSitemaps`. The `next-sitemap` package includes these in the sitemap index XML regardless of `generateRobotsTxt: false`. But no script or config ever *generates* these files — they're phantom references.

**Impact:** Google tries to fetch these sitemaps, gets 404s, and flags them as errors. This degrades Google's trust in your sitemap, causing it to deprioritize crawling your URLs. This is a likely contributor to the 159 "Discovered - currently not indexed" pages.

**Fix:** Remove phantom references from `robotsTxtOptions.additionalSitemaps`. Only `video-sitemap.xml` (which is actually generated by `scripts/generate-video-sitemap.ts`) should remain.

**File:** `next-sitemap.config.js` (included in this fix bundle)

---

### 2. Admin/Non-Indexable Pages in Sitemap (HIGH)

**Problem:** Pages like `/admin/media-upload` and `/chat` appear in `sitemap-0.xml`.

**Root Cause:** The `exclude` array in `next-sitemap.config.js` only excludes `/api/*` and `/404`. Admin pages, chat, drafts, and experiments are not excluded.

**Impact:** Wasted crawl budget on pages that return thin content or require authentication, diluting the crawl priority of your actual articles.

**Fix:** Add `/admin/*`, `/chat`, `/drafts/*`, `/experiments/*` to the `exclude` array.

**File:** `next-sitemap.config.js` (included in this fix bundle)

---

### 3. Duplicate H1 Tags on Every Article Page (HIGH)

**Problem:** Every article page has two `<h1>` elements with identical text. For example, on `/articles/abundance-fork`, there are two h1s both saying "The Abundance Fork: Post-Scarcity Utopia or Techno-Feudalism".

**Root Cause:** The article page template (`pages/articles/[slug].tsx`) renders the frontmatter `title` as an `<h1>` in the hero section. But the MDX content files (`pages/docs/articles/*.mdx`) also start with `# Title` — a Markdown h1 heading. When MDX renders, this becomes a second `<h1>`.

**Impact:** Multiple h1 tags confuse search engines about the primary topic of the page. While not a direct ranking penalty, it's a strong negative signal for content structure.

**Fix:** Run the included `scripts/fix-mdx-duplicate-h1.ts` script to strip the redundant `# Title` from all MDX files. The template's h1 in the hero section remains as the single, authoritative h1.

```bash
# Preview first (safe, no changes)
npx tsx scripts/fix-mdx-duplicate-h1.ts --dry-run

# Apply
npx tsx scripts/fix-mdx-duplicate-h1.ts
```

---

### 4. Articles Returning 404 (HIGH)

**Problem:** At least 10 article URLs in the sitemap return 404. Example: `/articles/ai-kill-switch` returns 404, but the actual file is `ai-kill-switch-postmortem.mdx` (slug: `ai-kill-switch-postmortem`).

**Root Cause:** Slug mismatches between what's in the sitemap and what's generated by `getStaticPaths`. This happens when articles are renamed without adding redirects.

**Fix (two-part):**

**Part A:** Run the audit script to identify all phantom URLs:
```bash
npx tsx scripts/audit-sitemap-404s.ts
```

**Part B:** For each phantom URL, either:
- Add a redirect in `next.config.js` (if the content moved to a new slug), OR
- Remove the old URL from Google's index via Search Console's URL Removal tool

Example redirect to add to `next.config.js`:
```js
{ source: '/articles/ai-kill-switch', destination: '/articles/ai-kill-switch-postmortem', permanent: true },
```

---

### 5. 108 Redirect Issues (MEDIUM — Self-Resolving)

**Problem:** 108 URLs show "Page with redirect" in Search Console.

**Root Cause:** The `next.config.js` has 301 redirects from `/docs/articles/:slug` to `/articles/:slug`. These are old URLs that were previously indexed. Google is correctly following the redirects but flagging them.

**Impact:** This is working as intended. The 301 redirects properly pass link equity. However, these flagged URLs waste crawl budget.

**Fix:** No code change needed — Google will eventually drop the old `/docs/articles/` URLs from the index. To speed this up:
1. Submit the updated sitemap (after fixing issues above) in Search Console
2. Optionally use Search Console's URL Removal tool to request removal of `/docs/articles/*` pattern

---

### 6. Robots.txt Improvements (MEDIUM)

**Problem:** Current robots.txt doesn't disallow admin, API, draft, or experiment paths. Also includes `Crawl-delay: 1` which is ignored by Google but may slow other crawlers unnecessarily.

**Fix:** Updated `public/robots.txt` included in this bundle. Key changes:
- Added `Disallow` for `/api/`, `/admin/`, `/drafts/`, `/experiments/`, `/_next/`
- Added hub pages to `Allow` directives for emphasis
- Removed `Crawl-delay` (Google ignores it; if you want to rate-limit other bots, keep it)

---

### 7. 107 Crawled But Not Indexed + 159 Discovered But Not Indexed (INDIRECT)

These are largely downstream effects of issues 1-6 above:

- **Broken sitemaps** reduce Google's confidence in your URL inventory
- **404s in the sitemap** waste crawl budget
- **Admin pages** dilute crawl priority
- **Duplicate h1s** signal poor content structure
- **Redirect chains** burn crawl budget on non-canonical URLs

After implementing fixes 1-6, resubmit your sitemap in Search Console and request re-indexing of your key pages. The indexing numbers should improve significantly within 2-4 weeks.

## Implementation Order

1. **`next-sitemap.config.js`** — Drop in the fixed version (this bundle)
2. **`public/robots.txt`** — Drop in the fixed version (this bundle)
3. **Run `fix-mdx-duplicate-h1.ts`** — Strip duplicate h1s from all MDX files
4. **Run `audit-sitemap-404s.ts`** — Identify 404 article URLs
5. **Add redirects** in `next.config.js` for any renamed articles found in step 4
6. **Deploy to Vercel** — Push changes, verify the build succeeds
7. **Verify in Search Console:**
   - Check `/sitemap.xml` — should only reference `sitemap-0.xml` and `video-sitemap.xml`
   - Check `/sitemap-core.xml` and `/sitemap-articles.xml` — should no longer appear
   - Submit updated sitemap for re-crawl
   - Use "URL Inspection" on a few article pages to verify single h1, proper meta tags
8. **Request indexing** for your highest-priority unindexed pages
